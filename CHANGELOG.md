## Versionamento

Este projeto segue o Semantic Versioning:

- **MAJOR** - Mudanças incompatíveis na API
- **MINOR** - Novas funcionalidades compatíveis
- **PATCH** - Correções de bugs

Exemplo: `1.2.3`
- 1 = MAJOR (versão principal)
- 2 = MINOR (funcionalidades)
- 3 = PATCH (correções)
